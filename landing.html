The Governance Stack

Three layers. One coherent architecture.

Most AI safety tooling lives where the agent can see it — and route around it. This stack doesn't.

LAYER 1 · SPEND
AEGIS
@rocketlang/aegis · npm · AGPL-3.0
Governs what the agent spends

Vendor-neutral kill-switch for AI agents. Hard budget caps, human heartbeat, unified cross-surface visibility. Born from a real $200 incident.

  • ▸ Hard budget cap per session / 5h / day / week
  • ▸ KAVACH DAN Gate — L1–L4 destructive command approval via Telegram / WhatsApp
  • ▸ Human heartbeat — pauses agents when you leave
  • ▸ Agent spawn governance — depth + concurrency limits
  • ▸ Unified dashboard — all surfaces, all sessions, live
  • ▸ Claude Code + OpenAI Codex + LangChain + n8n
LAYER 2 · BEHAVIOR
KavachOS
@rocketlang/kavachos · npm · AGPL-3.0
Governs what the agent does

seccomp-bpf + cgroup BPF kernel enforcement. The agent can only make the syscalls its trust level permits — nothing else reaches the kernel.

  • ▸ seccomp-bpf profiles from trust_mask integer
  • ▸ cgroup BPF egress — per-session network allowlist at connect4/6
  • ▸ Exec allowlist — first-hop binary whitelist per agent type
  • ▸ L7 TLS proxy — SQL / PII / EchoLeak firewall, BCC injection detection
  • ▸ SCMP_ACT_NOTIFY supervisor — live exec path resolution
  • ▸ Falco rules per domain: maritime · logistics · OT · finance
LAYER 3 · PROOF
PRAMANA
DOI 10.5281/zenodo.19273330 · AGPL-3.0 (EE)
Proves what happened

Cryptographic attestation of every decision AEGIS and KavachOS made. SHA-256 sealed, chained, tamper-evident. EU AI Act Article 14 evidence package.

  • ▸ SHA-256 sealed receipt per gate decision
  • ▸ Chained — each receipt commits the previous hash
  • ▸ `kavachos audit` verifies chain integrity offline
  • ▸ S3 anchoring for long-term custody (EE)
  • ▸ Merkle ledger for multi-session evidence bundles (EE)
  • ▸ EU AI Act Article 14 evidence package (Phase 5)
EU AI Act · Article 14

KavachOS is the airbag.
PRAMANA is the black box.
Article 14 requires both.

PRAMANA alone is just logging. It proves what happened but doesn't prevent the next bad thing.

KavachOS alone is just enforcement. It gates behavior but leaves no verifiable trail.

Together: the human can override via the HITL gate, and the override is recorded in a tamper-evident chain. That's the two halves nobody else is pairing.

EU AI Act Article 14 requires human oversight with evidence. Not one. Both. The deadline is 2 August 2026.

How the stack satisfies Article 14
🛡️
Human can intervene
KAVACH DAN Gate — every L3/L4 action pauses and awaits explicit approval. Silence defaults to STOP.
🔒
Decision is recorded
PRAMANA seals every ALLOW/DENY with SHA-256 + chain. The record cannot be altered without breaking the chain.
✅
Behavior is bounded
KavachOS blocks the syscall before it reaches the kernel. Not a log. Not an alert. A hard stop.

Install

AEGIS + KavachOS ship as npm packages. AGPL-3.0. No cloud dependency. All data stays local.
